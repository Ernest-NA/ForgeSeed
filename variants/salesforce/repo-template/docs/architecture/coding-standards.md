# Coding Standards

## General
- Prefer readability over cleverness.
- Keep Apex services cohesive.
- Avoid SOQL and DML inside loops.
- Enforce CRUD/FLS and sharing model consciously.
- Prefer existing project patterns over parallel implementations.

## Testing
- Add Apex tests for changed behavior.
- Use focused test data creation.
- Keep assertions specific.

## Review focus
- security
- governor limits
- bulkification
- naming consistency
- metadata impact
